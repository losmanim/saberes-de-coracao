## O que PODE mudar na sua vida

### 1. A forma como você se observa

Antes de mudança, vem a observação. Os saberes aqui presentes — especialmente epigenética e práticas contemplativas — oferecem ferramentas para perceber seus padrões:

- **Epigenética** — Você não é apenas seus genes. Suas escolhas (alimentação, sono, estresse, conexões) influenciam como seus genes se expressam. Isso significa que você tem mais agência do que pensava. Mas não é passe livre: é responsabilidade.

- **Gnose/Hermetismo** — "Conhece-te a ti mesmo." Não como autoconhecimento intelectual, mas como observação honesta de padrões, reações, automático.

- **Práticas** — Respiração consciente, meditação, mantras. Não são rituais místicos. São técnicas que alteram fisiologicamente e psicologicamente. Há neurologia por trás.

### 2. A forma como você responde ao estresse

Epigenética mostra que estresse crônico encurta telômeros, aumenta inflamação, desregula sistema imune. Respiração e meditação regulam ativamente o sistema nervoso autonômico (não é mito — há estudos de neuroimagem).

**Isto não significa** que se você ficar doente é porque não meditou. Significa que práticas de regulação fisiológica são aliadas na gestão do estresse.

### 3. A forma como você vê os outros

Um tema central em todos os saberes aqui: **conexão**. Tudo está conectado. Gnose, Hermetismo, Tao, Teosofia — todas chegam à mesma conclusão por caminhos diferentes:

**A separação é ilusão. A unidade é Real.**

Mas isto não é sobre "amar todo mundo" ou "ser iluminado." É sobre reconhecer que:

- Suas escolhas afetam o campo coletivo
- O que você faz aos outros, faz a si mesmo (lei de causa e efeito)
- Isolamento crônico tem custo biológico (epigenética confirma)

### 4. A forma como você encara a morte

Nenhuma tradição aqui promete vida eterna ou reencarnação garantida. O que oferece é:

- **Perspectiva de continuidade** — A consciência pode ser vista como fundamental (não-dual), não apenas emergente (materialista)
- **Não promete** — Apenas apresenta possibilidades para reflexão

O que isso muda? Para muitos, uma mudança de perspectiva sobre mortalidade reduz terror existencial e permite viver mais plenamente o agora.

## ⚙️ Documentação Técnica do Site

> Este guia explica como o site Saberes de Coração (v3.0) foi construído — arquitetura, componentes e lógica.

### Arquitetura Geral

O projeto segue uma arquitetura **data-driven**: todo o conteúdo está centralizado em um único arquivo JSON (`database/dados-unificados.json`), consumido pelo JavaScript para renderizar a interface dinamicamente.

```
database/dados-unificados.json  (fonte única de dados)
         │
         ▼
js/app.js                       (lógica: carrega, filtra, renderiza)
         │
         ├──► index.html         (página principal com cards dinâmicos)
         ├──► biblioteca.html    (página fixa com accordions)
         └──► css/style.css      (estilos visuais, tema claro/escuro)
```

### 📄 HTML — index.html (126 linhas)

A página principal é enxuta e semântica, dividida em:

| Seção | Função |
|-------|--------|
| **Header** | Logo, navegação, botões (aleatório, biblioteca, busca, tema) |
| **Hero** | Título, frase, estatísticas (carregadas via JS) |
| **Pilares** | Filtros por categoria: Todos, Espírito, Práticas, Ciência, Jornada, Vida Verdadeira, Multimídia |
| **Busca** | Campo de texto com filtro em tempo real |
| **Cards Grid** | Grid responsivo de cards (renderizado via JS) |
| **Modal** | Overlay com detalhes do saber (aberto ao clicar num card) |
| **Player** | Barra flutuante de áudio/vídeo (com controles, seek, volume) |
| **Footer** | Créditos e link para o código |

Características de acessibilidade implementadas:
- Roles ARIA (`banner`, `main`, `dialog`, `list`, `listitem`)
- `aria-label` em todos os botões e inputs
- `aria-pressed` nos botões de filtro
- `aria-expanded` no accordion da biblioteca
- `aria-live="polite"` nos resultados da busca
- `focus-visible` para navegação por teclado
- `sr-only` para textos ocultos para leitores de tela
- Focus trap no modal (Tab循环 preso dentro do modal)
- Suporte a Enter/Space para ativar cards

### 🎨 CSS — style.css (895 linhas)

**Variáveis CSS** (`:root`): Define cores, fontes, bordas e transições — facilita manutenção e temas.

**Tema escuro (padrão):** Fundo `#0d1117` (GitHub Dark), cards `#161b22`, texto `#c9d1d9`.

**Tema claro** (`.modo-claro`): Fundo `#f5f7fa`, cards brancos, texto escuro. Alternado via JS e salvo em `localStorage`.

**Cores por categoria** (usadas em bordas, badges e botões ativos):
| Categoria | Cor | Variável |
|-----------|-----|----------|
| 🌟 Todos | Azul destaque | `--cor-destaque: #58a6ff` |
| 🜂 Espírito | Roxo | `--cor-espirito: #9b59b6` |
| 🧠 Práticas | Azul | `--cor-praticas: #3498db` |
| 🔬 Ciência | Verde | `--cor-ciencia: #2ecc71` |
| 🧭 Jornada | Laranja | `--cor-jornada: #e67e22` |
| ∞ Vida Verdadeira | Vermelho | `--cor-vida: #e74c3c` |
| 🎵 Multimídia | Rosa | `--cor-midia: #e91e63` |

**Grid responsivo:** `grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))` — em telas < 600px vira 1 coluna.

**Modal:** Overlay com `backdrop-filter: blur(4px)`, animação de entrada (`modalIn`), tabelas, blocos de prática e citações estilizadas.

**Player fixo:** Barra flutuante no rodapé com animação `playerSlideUp`. Oculto por padrão (`hidden`), exibido ao reproduzir mídia.

**Animações:** Loading spinner (`spin`), entrada do modal, entrada do player.

### ⚡ JavaScript — app.js (771 linhas)

#### Estrutura Geral

```javascript
let dados = null;          // Dados carregados do JSON
let categoriaAtual = 'all'; // Filtro ativo
let ultimoElementoFocado;   // Para focus trap
```

#### Player (Objeto com métodos)

```javascript
const Player = {
    currentList: [],    // Lista atual (áudios ou vídeos)
    currentIndex: -1,   // Índice da faixa atual
    currentItem: null,  // Item atual
    isPlaying: false,   // Estado de reprodução
    ...
}
```

| Método | Função |
|--------|--------|
| `init()` | Configura eventos: play/pause, anterior/próximo, mute, seek, volume, atalhos de teclado (Espaço, P, N, M) |
| `buildList(tipo)` | Monta lista de mídia do JSON (áudio ou vídeo) |
| `open(tipo, id)` | Abre player e inicia reprodução |
| `togglePlay()` | Play/Pause |
| `prev()` / `next()` | Navegação entre faixas (cíclica) |
| `loadCurrent()` | Carrega faixa atual no player |
| `close()` | Fecha player e limpa |
| `toggleMute()` / `updateMuteIcon()` | Controle de volume |
| `formatTime(secs)` | Formata segundos para MM:SS |

**Atalhos de teclado** (quando player está visível):
- `Espaço` — Play/Pause
- `P` — Anterior
- `N` — Próximo
- `M` — Mudo

#### Funções Principais

| Função | Descrição |
|--------|-----------|
| `carregarDados()` | Fetch do `dados-unificados.json` e inicialização |
| `atualizarEstatisticas()` | Conta saberes, práticas e mídias |
| `renderizarSaberes(array)` | Gera HTML dos cards |
| `filtrarCategoria(cat)` | Filtra por pilar |
| `renderizarMidia()` | Exibe grid de áudios/vídeos |
| `buscarSaberes(termo)` | Busca por título, descrição ou tag |
| `abrirSaber(id)` | Abre modal com conteúdo completo do saber |
| `abrirMidia(tipo, id)` | Abre modal + player para mídia |
| `toggleBusca()` | Exibe/oculta campo de busca |
| `toggleTema()` | Alterna tema claro/escuro (salva em localStorage) |
| `saberAleatorio()` | Abre um saber aleatório |
| `configurarMidiaPath()` | Permite configurar caminho base dos arquivos de mídia |

#### Fluxo de Dados

```
1. DOMContentLoaded → Player.init(), carregar tema, carregarDados()
2. carregarDados() → fetch JSON → dados = response → atualizarEstatisticas() → renderizarSaberes()
3. Usuário clica filtro → filtrarCategoria() → renderizarSaberes() filtrado
4. Usuário busca → buscarSaberes() → renderizarSaberes() filtrado
5. Usuário clica card → abrirSaber() → modal com conteúdo do JSON
6. Usuário clica mídia → abrirMidia() → modal + Player.open()
```

### 🗄️ Banco de Dados — dados-unificados.json

**Versão:** 4.0 (24/05/2026) — 2476 linhas.

Estrutura:
```json
{
  "meta": { "versao", "atualizado", "descricao" },
  "categorias": [ { "id", "nome", "slug", "descricao", "cor", "icone" } ],
  "saberes": [ { "id", "categoria_id", "titulo", "descricao", "nivel",
                 "duracao", "tags", "fonte", "conteudo": {...},
                 "praticas": [...], "conexoes": [...] } ],
  "midia": {
    "audios": [ { "id", "titulo", "categoria", "tags", "arquivo",
                  "saberes_relacionados" } ],
    "videos": [ ... ]
  }
}
```

**Princípio:** Fonte única de dados (Single Source of Truth). Toda atualização de conteúdo deve ser feita neste JSON. O site, os ebooks e futuros componentes consomem daqui.
